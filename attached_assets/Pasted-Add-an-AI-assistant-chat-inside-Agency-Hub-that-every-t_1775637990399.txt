Add an AI assistant chat inside Agency Hub that every team member 
can use to get advice, ideas and support directly within the platform.
Do not modify anything that already exists.

---

OVERVIEW:
A dedicated AI chat powered by Claude (Anthropic API) accessible 
from anywhere in the app via a floating button or sidebar item.
The AI knows the context of the agency — clients, projects, campaigns, 
deadlines — and gives relevant, actionable advice based on real data.

---

- Icon: sparkle/star symbol

5. CONVERSATION MANAGEMENT

Each conversation is saved per user in Supabase:
- conversations: id, user_id, title (auto-generated from first message),
  context_type (general/project/client/report), context_id, 
  created_at, updated_at, message_count

- ai_messages: id, conversation_id, role (user/assistant), 
  content, tokens_used, feedback (positive/negative/null),
  created_at

Features:
- Conversations persist across sessions
- Auto-title: first 50 characters of the first user message
- User can rename, star or delete conversations
- Search across all past conversations
- "New conversation" button always visible
- Max 50 conversations stored per user (oldest auto-archived)

---

6. ANTHROPIC API INTEGRATION

Model: claude-sonnet-4-20250514
Max tokens: 2048 per response
Temperature: 0.7 (balanced between creative and factual)

API call structure (Next.js API route — server side only):
POST /api/ai/chat

Request body:
{
  messages: [...conversationHistory],
  context: {
    type: "project" | "client" | "report" | "general",
    data: { ...relevantContextData }
  }
}

System prompt built dynamically:
- Base agency context (always included)
- Page-specific context (injected when relevant)
- Current date and period
- Client/project data (when opened from those pages)

Streaming response:
- Use Anthropic streaming API so words appear progressively
- Show animated typing indicator (three dots) while waiting for first token
- User can stop generation mid-stream with a "Stop" button

Error handling:
- If API call fails show: "AI non disponibile al momento. Riprova tra poco."
- Rate limit handling with retry after delay
- Never expose API keys to the client side
